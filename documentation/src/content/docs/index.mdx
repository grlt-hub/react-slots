## Why React Slots?

React has no built-in slot system. When you build extensible apps — dashboards with plugins, host
shells with third-party modules — you fall back on prop drilling, ad-hoc context, or render props.
**React Slots** gives you named extension points and lets content be injected into them from
anywhere in your codebase.

## What you get

<CardGrid>
  <Card title="Inject from anywhere" icon="add-document">
    Plugins and separate packages call `insert()` to dock components into a slot — from anywhere, with no prop drilling
    and no coupling between the host and what fills it.
  </Card>
  <Card title="Control what renders" icon="list-format">
    Decide what shows and in what shape: `order` for render order, `filter` to gate rendering, and `mapProps` to reshape
    the props a slot passes down.
  </Card>
  <Card title="Know what's in a slot" icon="magnifier">
    React to slot contents with `useCount` and `usePresence` — e.g. render a heading only when the slot actually has
    visible content.
  </Card>
  <Card title="Type-safe & lightweight" icon="approve-check">
    Fully typed slot props out of the box, a runtime under ~2&nbsp;KB gzipped, and React 16.8 → 19 support.
  </Card>
</CardGrid>
